# 文档模板

> AGENTS.md 在生成 structure.md 和 tasks.md 时套用以下模板。
> 根据 project.md 中的「技术栈」字段自动选择对应模板。

---

## 课程类型识别规则

| 技术栈关键词 | 课程类型 |
|---|---|
| Spring Boot / Java / MySQL | 后端课程 |
| React / Vue / 前端 | 前端课程 |
| PHP | PHP 课程 |
| 微信小程序 / 小程序 | 小程序课程 |
| Windows / 桌面端 / PyQt / Electron | 桌面端课程 |
| 未注明 / AI 自动选定 | 默认按前端处理 |

---

## structure.md 模板

### 后端课程（Spring Boot / PHP）

```markdown
# 系统结构文档

## 项目信息
- 项目名称：
- 技术栈：
- 目标用户：

## 功能模块
### [模块名]
- 功能描述：
- 涉及接口：
- 输入 / 输出：

## 数据库表结构
### [表名]
| 字段名 | 类型 | 说明 |
|---|---|---|
| id | INT | 主键自增 |
| created_at | DATETIME | 创建时间 |

## API 接口列表
| 接口 | 方法 | 说明 |
|---|---|---|
| /api/[资源] | GET | 获取列表 |
| /api/[资源]/:id | GET | 获取详情 |
| /api/[资源] | POST | 新增 |
| /api/[资源]/:id | PUT | 修改 |

## 开发顺序
1. 搭建项目基础结构
2. 建库建表
3. [按功能模块顺序]
4. 联调测试主流程
```

## tasks.md 模板（通用）

```markdown
# 开发任务清单

> 项目：[项目名称]
> 状态说明：⬜ 未开始 / 🔄 进行中 / ✅ 已完成

---

- 状态：⬜
- 目标：[说明]
- 验收：项目可启动，无报错

### 任务 2：[第一个核心功能]
- 状态：⬜
- 目标：[说明]
- 验收：[用户可以完成什么操作]

### 任务 3：[第二个核心功能]
- 状态：⬜
- 目标：[说明]
- 验收：[用户可以完成什么操作]

### 任务 4：[第三个核心功能]
- 状态：⬜
- 目标：[说明]
- 验收：[用户可以完成什么操作]

- 验收：可以完整演示项目主流程

- 总任务：[N] 个

```
