# AI项目文档 — 校园二手交易平台

> 课程：PHP 开发
> 案例类型：自定义项目（列表展示 + 用户系统）
> 适用对象：AI 内部参照 + 学生查看示例

---

# 1. 项目概述

## 1.1 项目名称
校园二手交易平台

## 1.2 项目一句话定义
一个让学生发布和浏览校园二手物品的交易信息平台。

## 1.3 项目类型
Web 应用 / 信息发布平台

## 1.4 目标平台
桌面浏览器优先，手机浏览器可用

## 1.5 当前阶段
第一版 Demo 开发阶段

## 1.6 本次任务目标
先完成商品发布和列表浏览功能，再完成详情页

---

# 2. 项目目标与成功标准

## 2.1 项目目标
完成一个可演示的二手信息平台，支持发布、浏览和查看商品详情

## 2.2 用户价值
- 目标用户是：在校学生
- 他们当前的主要问题是：二手物品信息散落在各个群里，难以统一查找
- 这个项目提供的核心价值是：集中展示校园二手信息，方便买卖双方对接

## 2.3 成功标准
- 可以发布二手商品信息
- 可以浏览商品列表
- 可以查看商品详情
- 可以标记商品为已售出

---

# 3. 用户角色定义

## 3.1 角色列表
- 角色 1：卖家（发布商品）
- 角色 2：买家（浏览商品）
- 第一版不做登录，用昵称区分发布者

---

# 4. 核心问题与使用场景

## 4.1 当前问题
- 二手信息发在微信群里，时间久了找不到
- 不知道学校里有谁在卖什么

## 4.2 目标场景
- 学生毕业前发布闲置物品信息
- 其他同学浏览列表，找到感兴趣的商品联系卖家

## 4.4 最关键主流程
卖家填写商品信息（名称、价格、描述、联系方式）→ 提交发布
→ 商品出现在列表页
→ 买家浏览列表 → 点击进入详情页 → 查看联系方式

---

# 5. 功能范围

## 5.1 本期必须做
- 发布商品（名称、价格、描述、联系方式、分类）

## 5.3 本期明确不做

- 不做图片上传

---

# 6. 页面与模块清单

## 6.1 页面列表
- 页面 1：商品列表页（index.php）
- 页面 2：发布商品页（publish.php）
- 页面 3：商品详情页（detail.php）

## 6.2 每个页面的定义

### 页面：商品列表页
**主要内容：** 商品卡片列表（名称、价格、分类、状态标签）、分页控件
**主要操作：** 点击商品 → 进入详情页；点击「发布」→ 进入发布页

### 页面：发布商品页
**主要内容：** 发布表单（名称、价格、分类、描述、联系方式、昵称）
**主要操作：** 填写表单 → 提交 → 跳转列表页

### 页面：商品详情页
**主要内容：** 商品完整信息、联系方式、已售出标记按钮
**主要操作：** 点击「标记已售出」→ 状态更新

---

# 7. 数据结构定义

## 7.1 实体列表
- Item（商品）

## 7.2 字段定义

### 实体：Item

| 字段名 | 类型 | 是否必填 | 含义 | 示例值 |
|---|---|---|---|---|
| id | INT | 是 | 主键自增 | 1 |
| title | VARCHAR(100) | 是 | 商品名称 | "九成新自行车" |
| price | DECIMAL(10,2) | 是 | 价格 | 80.00 |
| category | VARCHAR(20) | 是 | 分类 | "数码/书籍/生活用品/其他" |
| description | TEXT | 否 | 商品描述 | "用了一年，无划痕" |
| contact | VARCHAR(50) | 是 | 联系方式 | "微信：xxx" |
| nickname | VARCHAR(50) | 是 | 发布者昵称 | "张同学" |
| status | VARCHAR(10) | 是 | 状态 | "在售" / "已售出" |
| created_at | DATETIME | 是 | 发布时间 | "2024-03-10" |

# 8. 业务规则

## 8.1 状态规则
- 商品状态只有两种：在售 / 已售出
- 状态不可逆（已售出不能改回在售，第一版保持简单）

## 8.2 数据规则
- 商品名称、价格、联系方式不能为空
- 价格必须为正数
- 列表页每页显示 10 条，按发布时间倒序

## 8.3 操作规则
- 发布成功后跳转到列表页
- 标记已售出后详情页显示「已售出」红色标签
