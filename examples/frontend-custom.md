# 1. 项目概述

## 1.1 项目名称
个人记账本

## 1.2 项目一句话定义
一个帮助用户记录每日收支、查看月度统计的个人财务管理应用。

## 1.3 项目类型
工具类应用 / Web 单页面

## 1.4 目标平台
桌面浏览器优先，移动端基础适配

## 1.5 当前阶段
第一版 Demo 开发阶段

## 1.6 本次任务目标
先完成收支记录的增删和列表展示，再完成月度统计功能

# 2. 项目目标与成功标准

## 2.1 项目目标
完成一个可演示的记账应用，支持记录收支和查看月度汇总

## 2.2 用户价值
- 目标用户是：需要管理个人财务的学生或个人
- 他们当前的主要问题是：不清楚每个月花了多少钱，钱花哪去了
- 这个项目提供的核心价值是：清晰记录每笔收支，直观展示月度统计

## 2.3 成功标准
- 可以新增收入 / 支出记录
- 可以查看记录列表
- 可以查看本月总收入、总支出、结余
- 页面可运行，无明显报错

## 2.4 演示标准
- 使用本地状态，不需要后端
- 可以演示记录 → 统计的完整流程

# 4. 核心问题与使用场景

## 4.1 当前问题
- 花了多少钱不清楚，月底才发现超支
- 不知道钱主要花在哪些类别上

## 4.2 目标场景
- 每次消费后立即记录
- 月底查看收支统计

## 4.3 高频使用场景
- 新增支出记录
- 查看本月记录列表
- 查看月度汇总数据

## 4.4 最关键主流程
用户打开页面
→ 填写金额、类别、备注
→ 选择收入或支出，点击添加
→ 列表显示新记录
→ 查看顶部统计卡片（本月收入 / 支出 / 结余）

# 7. 数据结构定义

## 7.1 实体列表
- Record（收支记录）

## 7.2 每个实体的字段定义

### 实体：Record

| 字段名 | 类型 | 是否必填 | 含义 | 示例值 |
|---|---|---|---|---|
| id | string | 是 | 唯一ID | "1710000001" |
| type | string | 是 | 类型：income/expense | "expense" |
| amount | number | 是 | 金额 | 25.5 |
| category | string | 是 | 类别 | "餐饮" / "交通" / "其他" |
| note | string | 否 | 备注 | "午饭" |
| createdAt | Date | 是 | 记录时间 | new Date() |

---

# 8. 业务规则

## 8.1 数据规则
- 金额必须为正数，不能为 0
- 金额保留两位小数
- 类别从固定列表选择（餐饮、交通、购物、娱乐、其他）

## 8.2 统计规则
- 总收入 = 所有 type 为 income 的金额之和
- 总支出 = 所有 type 为 expense 的金额之和
- 结余 = 总收入 - 总支出
- 结余为负数时显示红色

## 8.3 操作规则
- 列表按创建时间倒序显示
- 删除无需二次确认（第一版保持简单）
- 添加成功后表单自动清空

# 9. 界面与交互要求

## 9.1 整体风格
简洁清晰，收入用绿色、支出用红色区分

## 9.2 布局要求
- 顶部三个统计卡片并排
- 表单和列表上下排列
- 每条记录左侧显示类别图标，右侧显示金额

## 9.3 颜色要求
- 收入金额：绿色
- 支出金额：红色
- 结余为负：红色显示

## 9.4 反馈要求
- 金额为空或不合法时，输入框提示错误
- 添加成功后表单清空

# 10. 技术约束

## 10.1 技术栈要求
- React（或 Vue 3）
- 样式使用 CSS 或 Tailwind CSS
- 数据使用组件内 state

## 10.2 实现原则
- 统计用 useMemo 或 computed 计算，不手动维护

## 10.3 复杂度限制
- 不引入状态管理库
- 不做路由
