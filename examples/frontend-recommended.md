# 1. 项目概述

## 1.1 项目名称
个人任务清单 Todo App

## 1.2 项目一句话定义
一个帮助用户记录、管理和追踪每日待办事项的单页面应用。

## 1.3 项目类型
工具类应用 / Web 单页面

## 1.4 目标平台
桌面浏览器优先，移动端基础适配

## 1.5 当前阶段
第一版 Demo 开发阶段

## 1.6 本次任务目标
先完成任务的增删改查和状态切换，再完成分类筛选功能

# 2. 项目目标与成功标准

## 2.1 项目目标
完成一个可演示的最小可用版本，支持任务管理的完整交互

## 2.2 用户价值
- 目标用户是：需要管理日常任务的个人用户
- 他们当前的主要问题是：任务记在备忘录里没有完成状态，容易忘记或重复做
- 这个项目提供的核心价值是：清晰的任务列表 + 完成状态追踪

## 2.3 成功标准
- 可以新增、删除任务
- 可以标记任务为已完成 / 未完成
- 可以按状态筛选任务（全部 / 未完成 / 已完成）
- 页面可运行，无明显报错

## 2.4 演示标准
- 使用本地状态，不需要后端接口
- 可以演示新增、完成、删除、筛选的完整交互
- 页面布局清晰，便于课堂展示

# 4. 核心问题与使用场景

## 4.1 当前问题
- 任务随手记在手机备忘录，没有完成状态
- 不知道今天还有哪些事没做完
- 已完成的任务和未完成的混在一起

## 4.2 目标场景
- 早上打开 App 查看今天的待办事项
- 完成一项后打勾标记
- 查看还有哪些未完成

## 4.3 高频使用场景
- 新增任务
- 标记完成
- 查看未完成列表

## 4.4 最关键主流程
用户打开页面
→ 看到任务列表
→ 输入新任务并添加
→ 完成某个任务后点击勾选
→ 筛选查看「未完成」任务

## 5.1 本期必须做

# 6. 页面与模块清单

## 6.1 页面列表
- 页面 1：主页面（唯一页面）

## 6.2 每个页面的定义

### 页面：主页面
**页面作用：** 展示任务列表，完成所有操作
**主要内容：**
- 顶部标题
- 任务输入框 + 添加按钮
- 状态筛选标签（全部 / 未完成 / 已完成）
- 任务列表（每项含勾选框、任务名、删除按钮）
- 底部统计（未完成 N 项）

**主要操作：**
- 输入任务名 → 点击添加
- 点击勾选框 → 切换完成状态
- 点击删除 → 移除任务
- 点击筛选标签 → 切换显示范围

## 6.3 模块列表
- 任务输入组件
- 筛选标签组件
- 任务列表组件
- 任务项组件（含勾选、文字、删除）

# 7. 数据结构定义

## 7.1 实体列表
- Task（任务）

## 7.2 每个实体的字段定义

### 实体：Task

| 字段名 | 类型 | 是否必填 | 含义 | 示例值 |
|---|---|---|---|---|
| id | string | 是 | 唯一ID（时间戳生成） | "1710000001" |
| text | string | 是 | 任务内容 | "完成作业" |
| completed | boolean | 是 | 是否完成 | false |
| createdAt | Date | 是 | 创建时间 | new Date() |

## 7.3 实体关系说明
- 只有 Task 一个实体，存放在组件 state 中

---

# 8. 业务规则

## 8.1 状态规则
- 任务状态只有两种：完成 / 未完成
- 点击勾选框切换状态，可以反复切换

## 8.2 数据规则
- 任务内容不能为空（输入框为空时禁止添加）
- 任务内容前后空格自动去除
- 第一版数据存在内存中，刷新后重置

## 8.3 操作规则
- 按 Enter 键也可以添加任务
- 删除操作无需二次确认
- 筛选只影响展示，不影响数据

# 9. 界面与交互要求

## 9.1 整体风格
简洁清爽，重点突出任务列表，不追求复杂视觉

## 9.2 布局要求
- 内容居中，最大宽度 600px
- 输入框和添加按钮在同一行
- 已完成任务文字加删除线

## 9.3 颜色要求
- 白底为主
- 主按钮用蓝色
- 已完成任务文字变灰

## 9.4 反馈要求
- 输入框为空点击添加时，输入框抖动或边框变红
- 已完成任务加删除线，视觉区分明显

---

# 10. 技术约束

## 10.1 技术栈要求
- React（或 Vue 3）
- 样式使用 CSS Modules 或 Tailwind CSS
- 数据使用组件内 state，不接后端

## 10.2 实现原则
- 优先使用 useState 管理状态，不引入 Redux / Vuex
- 组件拆分清晰但不过度
- 非必要不引入额外依赖

## 10.3 复杂度限制
- 不引入复杂状态管理库
- 不做路由
- 不做动画库
