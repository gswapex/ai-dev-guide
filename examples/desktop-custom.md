# 2. 项目目标与成功标准

## 2.1 项目目标
完成一个可运行的番茄钟工具，支持专注计时和短暂休息的循环

## 2.2 用户价值
- 目标用户是：需要提高学习或工作专注力的个人用户
- 他们当前的主要问题是：容易分心，不知道自己专注了多长时间
- 核心价值：结构化的专注时间管理，帮助保持节奏感

## 2.3 成功标准
- 可以开始 25 分钟专注计时
- 计时结束后弹出提示，自动切换到 5 分钟休息
- 可以暂停和重置计时器
- 可以查看今日完成的番茄数

## 2.4 演示标准
- 程序可以启动运行
- 计时器可以正常倒计时
- 时间结束有声音或弹窗提示

# 4. 核心问题与使用场景

## 4.1 当前问题
- 学习时容易被手机或其他事情打断
- 不知道自己每天真正专注了多长时间

## 4.2 目标场景
- 开始学习前启动番茄钟
- 25 分钟后提示休息
- 休息结束后继续下一个番茄

## 4.4 最关键主流程
用户打开软件 → 点击「开始专注」
→ 倒计时 25 分钟
→ 时间到弹出提示「专注结束，休息一下！」
→ 自动开始 5 分钟休息倒计时
→ 休息结束提示，今日番茄数 +1
→ 用户继续开始下一个番茄

# 8. 业务规则

## 8.1 计时规则
- 专注时长：25 分钟（默认，可后期配置）
- 休息时长：5 分钟（默认）
- 专注结束 → 自动切换到休息
- 休息结束 → 提示开始下一个，不自动开始

## 8.2 状态规则
- 计时器状态：未开始 / 进行中 / 已暂停 / 已结束
- 暂停后可以继续，不重置进度
- 重置后回到专注初始状态

## 8.3 数据规则
- 每完成一个完整专注番茄（25分钟），今日番茄数 +1
- 暂停或中途重置不计入
- 今日番茄数按日期存本地，每天清零

# 10. 技术约束

## 10.1 技术栈要求
- Electron 或 PyQt5 / PyQt6
- 计时器用系统定时器实现
- 数据保存为本地 JSON 文件

## 10.2 实现原则
- 计时器用 setInterval（Electron）或 QTimer（PyQt）实现

## 10.3 复杂度限制
- 不做系统托盘
