# AI项目文档 — 班级投票小程序

> 课程：微信小程序开发
> 案例类型：自定义项目（数据统计 + 结果展示）
> 适用对象：AI 内部参照 + 学生查看示例

---

# 1. 项目概述

## 1.1 项目名称
班级投票小程序

## 1.2 项目一句话定义
一个让班长发起投票、同学匿名参与并实时查看结果的微信小程序。

## 1.3 项目类型
小程序 / 工具类应用

## 1.4 目标平台
微信小程序（手机端）

## 1.5 当前阶段
第一版 Demo 开发阶段

## 1.6 本次任务目标
完成投票创建、参与和结果查看功能

---

# 2. 项目目标与成功标准

## 2.1 项目目标
完成可演示的投票流程，支持创建投票、匿名参与、实时查看结果

## 2.2 用户价值
- 目标用户是：班级同学
- 他们当前的主要问题是：举手投票不方便，结果不透明
- 核心价值：匿名线上投票，结果公开透明

## 2.3 成功标准
- 可以创建投票（标题 + 选项）
- 可以参与投票（每人只能投一次）
- 可以查看各选项票数和占比

---

# 3. 用户角色定义

## 3.1 角色列表
- 角色 1：发起者（创建投票）
- 角色 2：参与者（投票）
- 第一版不做登录，用微信 openId 区分用户

---

# 4. 核心问题与使用场景

## 4.4 最关键主流程
发起者填写投票标题和选项 → 创建投票
→ 分享给同学
→ 同学打开小程序 → 选择选项 → 提交
→ 查看实时投票结果

---

# 5. 功能范围

## 5.1 本期必须做
- 创建投票（标题 + 2～4个选项）
- 参与投票（单选）
- 防重复投票（同一用户只能投一次）
- 查看投票结果（票数 + 百分比进度条）

## 5.2 本期可选
- 投票列表页（查看所有投票）

## 5.3 本期明确不做
- 不做真实微信登录（用随机 ID 模拟）
- 不做投票截止时间
- 不做多选投票
- 不做投票删除
- 不做评论

---

# 6. 页面与模块清单

## 6.1 页面列表
- 页面 1：投票列表页（首页）
- 页面 2：创建投票页
- 页面 3：参与投票页
- 页面 4：投票结果页

## 6.2 每个页面的定义

### 页面：投票列表页
**主要内容：** 所有投票的标题列表
**主要操作：** 点击投票 → 进入参与页或结果页；点击「创建」→ 创建投票

### 页面：创建投票页
**主要内容：** 投票标题输入框、选项输入框（最多4个）
**主要操作：** 填写 → 提交 → 跳转列表页

### 页面：参与投票页
**主要内容：** 投票标题、选项列表（单选）
**主要操作：** 选择选项 → 提交 → 跳转结果页

### 页面：投票结果页
**主要内容：** 各选项票数 + 百分比进度条 + 总票数

---

# 7. 数据结构定义

## 7.1 实体列表
- Poll（投票）
- VoteRecord（投票记录）

## 7.2 字段定义

### 实体：Poll

| 字段名 | 类型 | 含义 |
|---|---|---|
| _id | string | 文档ID |
| title | string | 投票标题 |
| options | array | 选项列表 [{text: string, count: number}] |
| createdAt | date | 创建时间 |

### 实体：VoteRecord

| 字段名 | 类型 | 含义 |
|---|---|---|
| _id | string | 文档ID |
| pollId | string | 关联投票ID |
| optionIndex | number | 所选选项序号 |
| userId | string | 用户标识（随机ID存本地） |
| createdAt | date | 投票时间 |

---

# 8. 业务规则

## 8.1 防重复规则
- 用 wx.getStorageSync 存储已投票记录
- 已投票用户访问参与页时直接跳转结果页

## 8.2 数据规则
- 投票标题不能为空
- 选项最少 2 个，最多 4 个
- 选项内容不能为空

---

# 9. 界面与交互要求

## 9.1 整体风格
简洁，结果页用进度条直观展示占比

## 9.2 反馈要求
- 投票成功后自动跳转结果页
- 已投票提示「你已参与过此投票」
