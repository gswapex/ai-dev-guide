# AI项目文档 — 学生成绩管理系统

> 课程：Spring Boot / Java 后端
> 案例类型：推荐项目
> 适用对象：AI 内部参照 + 学生查看示例

---

# 1. 项目概述

## 1.1 项目名称
学生成绩管理系统

## 1.2 项目一句话定义
一个帮助老师录入、查询和统计学生考试成绩的后端管理系统。

## 1.3 项目类型
管理系统 / Web 应用

## 1.4 目标平台
桌面浏览器优先，PC 端使用

## 1.5 当前阶段
第一版 Demo 开发阶段

## 1.6 本次任务目标
先完成核心数据结构和 CRUD 接口，再完成统计查询接口

---

# 2. 项目目标与成功标准

## 2.1 项目目标
完成一个可演示的最小可用版本，支持成绩录入、查询和基础统计

## 2.2 用户价值
- 目标用户是：任课老师
- 他们当前的主要问题是：用 Excel 手动录成绩、算平均分，容易出错且效率低
- 这个项目提供的核心价值是：系统化录入和自动统计，减少重复劳动

## 2.3 成功标准
- 可以录入学生信息和考试成绩
- 可以查询某个学生的所有成绩
- 可以统计班级平均分、最高分、不及格人数
- 接口无明显报错，数据流转符合预期

## 2.4 演示标准
- 使用假数据即可，不强制接真实数据库
- 可以演示新增、查询、统计三个核心流程
- 接口返回结构清晰，便于课堂讲解

# 3. 用户角色定义

## 3.1 角色列表
- 角色 1：老师（主要操作者）
- 角色 2：管理员（第一版暂不区分，与老师权限相同）

## 3.2 每个角色的权限定义

### 角色：老师

**身份说明：** 任课老师，负责录入和查询成绩

**可以做的事：**
- 新增学生信息
- 录入考试成绩
- 查询学生成绩列表
- 查看班级统计数据

**不可以做的事：**
- 删除学生记录（第一版不做）
- 修改其他老师录入的数据（第一版不做权限隔离）

---

# 4. 核心问题与使用场景

## 4.1 当前问题
- 老师用 Excel 手动录成绩，列多了容易搞混
- 统计平均分、不及格人数要手动写公式，耗时
- 数据分散，不方便历史查询

## 4.2 目标场景
- 考试结束后老师集中录入全班成绩
- 老师快速查询某个学生的历次成绩
- 老师查看班级整体成绩分布

## 4.3 高频使用场景
- 录入成绩
- 查询学生成绩列表
- 查看班级统计

## 4.4 最关键主流程
老师打开系统
→ 新增本次考试信息（科目、日期）
→ 逐个录入学生成绩
→ 查看班级统计结果（平均分、最高分、不及格人数）

# 7. 数据结构定义

## 7.1 实体列表
- Student（学生）
- Exam（考试）
- Score（成绩记录）

## 7.2 每个实体的字段定义

### 实体：Student

| 字段名 | 类型 | 是否必填 | 含义 | 示例值 |
|---|---|---|---|---|
| id | Long | 是 | 主键自增 | 1 |
| name | String | 是 | 学生姓名 | "张三" |
| studentNo | String | 是 | 学号 | "2024001" |
| className | String | 是 | 班级 | "计算机1班" |
| createdAt | DateTime | 是 | 创建时间 | "2024-03-01" |

### 实体：Exam

| 字段名 | 类型 | 是否必填 | 含义 | 示例值 |
|---|---|---|---|---|
| id | Long | 是 | 主键自增 | 1 |
| subject | String | 是 | 考试科目 | "Java基础" |
| examDate | Date | 是 | 考试日期 | "2024-03-10" |
| createdAt | DateTime | 是 | 创建时间 | "2024-03-10" |

### 实体：Score

| 字段名 | 类型 | 是否必填 | 含义 | 示例值 |
|---|---|---|---|---|
| id | Long | 是 | 主键自增 | 1 |
| studentId | Long | 是 | 关联学生ID | 1 |
| examId | Long | 是 | 关联考试ID | 1 |
| score | Double | 是 | 分数 | 85.5 |
| createdAt | DateTime | 是 | 创建时间 | "2024-03-10" |

## 7.3 实体关系说明
- 一个 Student 可以有多条 Score 记录
- 一个 Exam 可以对应多条 Score 记录
- 每条 Score 必须绑定一个 Student 和一个 Exam

# 8. 业务规则

## 8.1 状态规则
- 成绩分数范围：0 ～ 100
- 60分以下视为不及格

## 8.2 权限规则
- 第一版不做权限区分，所有人可操作

## 8.3 数据规则
- 学生姓名和学号不能为空
- 同一学生同一考试不能重复录入成绩
- 创建时间自动生成，不需要手动填写

## 8.4 操作规则
- 保存成功后返回成功提示
- 查询列表默认按创建时间倒序排列
- 统计数据实时根据已录入成绩计算
