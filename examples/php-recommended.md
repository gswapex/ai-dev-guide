# 1. 项目概述

## 1.1 项目名称
在线投票系统

## 1.2 项目一句话定义
一个让发起者创建投票、参与者提交选项并实时查看结果的网页应用。

## 1.3 项目类型
Web 应用

## 1.4 目标平台
桌面浏览器优先，手机浏览器可用

## 1.5 当前阶段
第一版 Demo 开发阶段

## 1.6 本次任务目标
先完成投票创建和参与功能，再完成结果统计展示

# 2. 项目目标与成功标准

## 2.1 项目目标
完成一个可演示的投票流程，支持创建投票、参与投票、查看结果

## 2.2 用户价值
- 目标用户是：需要快速收集意见的班级、社团或小组
- 他们当前的主要问题是：举手投票不透明，微信群里问意见容易混乱
- 这个项目提供的核心价值是：线上投票流程清晰，结果实时可查

## 2.3 成功标准
- 可以创建投票（标题 + 选项）
- 可以参与投票（每人只能投一次）
- 可以查看投票结果（各选项票数和占比）

## 2.4 演示标准
- 可以演示创建 → 投票 → 查看结果的完整流程
- 使用假数据或真实数据库均可

# 3. 用户角色定义

## 3.1 角色列表

# 4. 核心问题与使用场景

## 4.1 当前问题
- 班级举手投票不透明，容易有争议
- 微信群里问意见，回复乱且难统计

## 4.2 目标场景
- 班长创建一个投票，分享链接给同学
- 同学点开链接参与投票
- 投票结束后查看结果

## 4.4 最关键主流程
发起者填写投票标题和选项 → 提交创建
→ 系统生成投票页面链接
→ 参与者打开链接，选择选项，提交
→ 查看结果页面，看到各选项票数和百分比

# 5. 功能范围

## 5.1 本期必须做
- 创建投票（标题 + 2～5个选项）
- 参与投票（选择一个选项提交）
- 查看投票结果（各选项票数 + 占比）
- 防重复投票（同一浏览器只能投一次）

## 5.2 本期可选
- 投票截止时间设置
- 投票结果实时刷新

## 5.3 本期明确不做
- 不做用户登录和注册
- 不做投票删除
- 不做多选投票
- 不做评论功能
- 不做消息通知

# 6. 页面与模块清单

## 6.1 页面列表
- 页面 1：创建投票页（index.php）
- 页面 2：参与投票页（vote.php）
- 页面 3：投票结果页（result.php）

## 6.2 每个页面的定义

### 页面：创建投票页
**主要内容：** 投票标题输入框、选项输入框（2～5个）、提交按钮
**主要操作：** 填写表单 → 提交 → 跳转到投票页

### 页面：参与投票页
**主要内容：** 投票标题、选项列表（单选）、提交按钮
**主要操作：** 选择选项 → 提交 → 跳转到结果页

### 页面：投票结果页
**主要内容：** 投票标题、各选项票数、百分比进度条

# 7. 数据结构定义

## 7.1 实体列表
- Poll（投票）
- Option（选项）
- Vote（投票记录）

## 7.2 字段定义

### 实体：Poll

| 字段名 | 类型 | 是否必填 | 含义 |
|---|---|---|---|
| id | INT | 是 | 主键自增 |
| title | VARCHAR(200) | 是 | 投票标题 |
| created_at | DATETIME | 是 | 创建时间 |

### 实体：Option

| 字段名 | 类型 | 是否必填 | 含义 |
|---|---|---|---|
| id | INT | 是 | 主键自增 |
| poll_id | INT | 是 | 关联投票ID |
| content | VARCHAR(100) | 是 | 选项内容 |

### 实体：Vote

| 字段名 | 类型 | 是否必填 | 含义 |
|---|---|---|---|
| id | INT | 是 | 主键自增 |
| poll_id | INT | 是 | 关联投票ID |
| option_id | INT | 是 | 所选选项ID |
| voter_token | VARCHAR(64) | 是 | 浏览器标识（防重复） |
| created_at | DATETIME | 是 | 投票时间 |

## 7.3 实体关系说明
- 一个 Poll 有多个 Option
- 一个 Option 可以有多条 Vote 记录
- voter_token 用 session 或 cookie 生成，防止同一浏览器重复投票

# 8. 业务规则

## 8.1 状态规则
- 投票创建后立即可用，无需审核
- 第一版不做截止时间，投票长期有效

## 8.2 防重复规则
- 每个浏览器对同一投票只能投一次
- 用 SESSION 或 COOKIE 记录已投票状态
- 已投票再访问投票页时直接跳转到结果页

## 8.3 数据规则
- 投票标题不能为空
- 选项数量最少 2 个，最多 5 个
- 选项内容不能为空

# 10. 技术约束

## 10.1 技术栈要求
- PHP 原生（不强制使用框架）
- MySQL 数据库
- 前端使用原生 HTML + CSS，不引入前端框架

## 10.2 实现原则
- 优先使用最简单的实现
- 用 PDO 连接数据库
- 防 SQL 注入（使用预处理语句）

## 10.3 复杂度限制
- 不使用 Laravel / Symfony 等框架
- 不引入前端框架
- 不做 AJAX，用传统表单提交
