# 小米之家门店管理系统 — 开发任务拆分文档

> Spring Boot + React | 前后端分离 | V1.0

---

## 一、项目总览与分期规划

本项目共划分为三个开发阶段，建议按照阶段顺序推进。每个阶段完成后均可形成可运行的局部系统，降低开发风险。

| 阶段 | 名称 | 目标 | 预计工期 | 优先级 |
|------|------|------|----------|--------|
| 阶段一 | 基础框架 + 用户认证 | 项目骨架搭建完成，登录/注册可运行 | 3～5 天 | 高 |
| 阶段二 | 核心业务模块 | 商品、库存、订单、会员四大模块完成 | 10～15 天 | 高 |
| 阶段三 | 统计分析 + 系统完善 | 数据看板、报表、权限控制完成 | 5～7 天 | 中 |

---

## 二、阶段一：基础框架 + 用户认证

### 2.1 任务列表

| 序号 | 任务名称 | 所属层 | 说明 | 难度 |
|------|----------|--------|------|------|
| 1-1 | 初始化 Spring Boot 项目 | 后端 | 创建项目骨架，配置 pom.xml 依赖（Web/JPA/MySQL/JWT） | 低 |
| 1-2 | 配置 application.yml | 后端 | 配置数据库连接、端口 8080、跨域允许 3000 | 低 |
| 1-3 | 建立数据库与初始化表 | 数据库 | 执行 schema.sql，建立 USER_INFO / ROLE_INFO / PERMISSION_INFO 表 | 低 |
| 1-4 | 实现统一返回格式 Result\<T\> | 后端 | 封装 statusCode/opCode/opDesc/localTime/data 字段 | 低 |
| 1-5 | 全局异常处理器 | 后端 | 编写 GlobalExceptionHandler，捕获参数错误/业务异常/系统异常 | 中 |
| 1-6 | 实现用户注册接口 | 后端 | POST /v1/users/register，校验邮箱格式、密码强度，BCrypt 加密存储 | 中 |
| 1-7 | 实现用户登录接口 | 后端 | POST /v1/auth/login，验证账号密码，生成并返回 JWT Token | 中 |
| 1-8 | 实现 JWT 拦截器 | 后端 | 解析 Authorization Header，验证 Token 有效性，注入用户上下文 | 中 |
| 1-9 | 实现登出接口 | 后端 | POST /v1/auth/logout，将 Token 加入黑名单 | 中 |
| 1-10 | 初始化 React 项目 | 前端 | 创建项目，配置 axios、react-router，建立目录结构 | 低 |
| 1-11 | 封装 api/request.js | 前端 | 配置 baseURL=8080，请求/响应拦截器，401 跳转登录 | 低 |
| 1-12 | 开发登录页面 | 前端 | 表单校验、调用登录接口、存储 token、跳转首页 | 低 |
| 1-13 | 配置路由守卫 | 前端 | 未登录跳转 /login，角色权限路由隔离 | 中 |
| 1-14 | 开发整体布局组件 | 前端 | 侧边栏 Sidebar + 顶部栏 + 内容区 Layout | 低 |

### 2.2 阶段一验收标准

- ✅ 后端服务在 8080 端口正常启动，控制台无报错
- ✅ 使用 Postman 发送 POST /v1/auth/login，可获得 JWT Token
- ✅ 携带有效 Token 请求其他接口，返回 200；不携带返回 401
- ✅ 前端页面在 3000 端口正常运行，登录表单可提交
- ✅ 登录成功后跳转首页，刷新后登录状态保持
- ✅ 访问需要权限的路由时，未登录会自动重定向至登录页

## 三、阶段二：核心业务模块

| 序号 | 任务名称 | 接口/路径 | 说明 | 难度 |

### 3.5 阶段二验收标准

- ✅ 商品可完整增删改查，上下架状态可切换
- ✅ 入库/出库接口正确更新 product.stock，并生成日志记录
- ✅ 库存低于预警值时，前端列表中该商品行有红色标记
- ✅ 创建订单后库存正确减少，库存不足时接口返回明确错误
- ✅ 支付订单后，如关联会员则自动增加积分
- ✅ 会员积分累积后，等级字段自动更新
- ✅ 所有列表页分页正常工作
