# 指令：/ai-dev-start

直接套用已选定的目标项目，或从内置项目库中选择一个项目模板包，快速启动项目。

适合已经不需要从零引导、希望基于明确目标或项目库模板快速开始的用户。

## 你的任务

不要再走“从麻烦开始”的完整引导流程。

如果当前对话里还没有确认课程背景，先执行 `references/guide.md` 中的「第0阶段：确认课程背景」。

然后按以下顺序处理：

1. 确认用户是否已经给出目标项目
2. 如果未给出目标，则根据课程类型列出 `projects/*` 项目库中可用的项目目录供用户选择
3. 把 `projects/*` 视为可扩展的项目库：每个子目录都是一个项目模板包，不假设只存在一个项目
4. 如果选中的项目在 `projects/*` 中存在完整项目包，则优先读取该目录下的需求、接口、数据库、原型、任务拆分等核心文件
5. 整理成当前项目目标后，询问用户是否直接开始初始化开发文档
6. 用户确认后，转入 `/ai-dev-init`

## 行为规则（必须遵守）

- 一次只问一个问题，等待用户回答后再继续
- 问题用口语，不用技术术语
- 每个问题提供 2～3 个选项，用户也可以自由回答
- 用户回答后先确认，再推进下一步
- 不要提前跳到写代码
- 不要重复 `/ai-dev-guide` 的完整需求推导流程
- 优先复用 `projects/*` 中的完整项目包
- 把 `projects/*` 视为项目库，而不是单个案例目录
- 不要把 `examples/*.md` 当成 `/ai-dev-start` 的默认输入源
- 全程用中文

## 启动语

直接说：

> 好，我们直接开始。
> 你这次想直接做哪个项目？
> 如果你已经有目标，我就按这个目标开始；
> 如果还没定，我可以根据课程方向，从内置项目库里给你一个可直接套用的项目列表。
